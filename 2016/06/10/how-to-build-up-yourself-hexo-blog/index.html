_config.yml采用的是YAML文件格式，数据采用key:value的格式，有一点需要注意的是，key和value之间必须有一个空格，否则会导致格式不正确。

关于主题的设置

更换Hexo的主题非常简单，将项目clone到themes文件夹，然后将根目录下_config.yml得theme切换为新的主题的名字即可。

除此之外，每种主题都可以自由的配置一些参数和特性，修改特定主题的属性时，需要修改主题所在文件夹下的配置文件_config.yml，注意区别于根目录下的全局配置文件。

常用命令

hexo自带的命令非常简洁，学习起来也很简单。

常用的命令就下面几个

  • hexo new title 生成一篇名为“title”的新文章，采用默认的模板
  • hexo g 重新渲染Blog
  • hexo s 启动本地服务器，默认采用的网文地址是：http://localhost:4000/,可以用来本地测试
  • hexo d 将本地public目录下的文件内容提交到Github，具体的项目地址在_config.yml设置
  • hexo clean 清空本地的缓存文件db.json和已生成的静态文件public

一般来说，一个正常的工作流是这样的：hexo new一篇模板文章，写作完成后，hexo g进行渲染，hexo s在本地试运行，本地测试没问题后，hexo d将生成的内容部署到Github对应的项目地址，完成文章的发表。

使用Maupassant主题过程中的坑

下面是在使用Maupassant主题时遇到的几个问题

开启了TOC不显示目录

在Maupassant主题中，可以通过toc来设置显示目录结构，这是我比较喜欢的Feature。但是只支持使用“===”和“—”作为标题等级的识别符，不支持使用“#”方式。

npm install hexo-renderer-sass出现问题

如果这一步出现问题，那么就会出现修改了主题以后，界面没有样式的问题，修复方法就是：

  • npm uninstall hexo-renderer-sass –save 将sass卸载
  • 安装cpnm，具体安装方法自行搜索
  • 使用cnpm install hexo-renderer-sass –save 重新安装saas即可

部署时CNAME文件被删除

如果你想绑定个人域名的话，就需要完成CNAME解析，在项目的根目录放置CNAME文件，但是直接在Github项目的根目录添加文件，在下次更新的时候，会自定删除CNAME文件，因为每次public文件都是再次生成的，CNAME文件会丢失。

解决方案就是将CNAME文件放置在根目录source文件夹里面，在生成public文件的时候，hexo会自动将CNAME文件拷贝过去。

添加百度统计

如果你想添加统计工具，观察每天博客的流量情况，那么就可以申请百度统计，把申请得到的id填写在themes/maupassant/_config.yml对应位置即可。

添加Disqus评论

相对多说来说，我更喜欢Disqus，如果要使用Disqus，需要去官网申请开通。然后将开通时设置的user_name配置到themes/maupassant/_config.yml即可。开通地址请戳：https://disqus.com/admin/create/
